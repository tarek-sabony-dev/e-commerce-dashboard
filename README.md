# E-commerce Dashboard

A Next.js 15 + TypeScript dashboard scaffold for an e-commerce admin UI. This README gives a clear picture of the current project state, file structure, what each major file does, and how the frontend expects to integrate with your backend.

## Overview

- **Framework:** Next.js 15 (Turbopack for dev/build)
- **Language:** TypeScript 5
- **React:** 19
- **Styling:** Tailwind CSS v4 (via PostCSS plugin)
- **Linting:** ESLint 9 with Next presets
- **State & Forms:** Redux Toolkit, React Redux, React Hook Form, Zod
- **UI/UX:** Radix UI, shadcn/ui config, lucide-react icons, next-themes
- **Files/Uploads:** EdgeStore client/server packages present

## Key Files & Directories

### Configuration Files

- **`components.json`** - Generated by shadcn, contains configuration options for UI components and styling
- **`eslint.config.mjs`** - ESLint configuration
- **`next.config.ts`** - Next.js configuration
- **`postcss.config.mjs`** - PostCSS configuration
- **`tsconfig.json`** - TypeScript configuration

### Public Directory

- **`public/`** - Contains static images that are used as snapshots and thumbnails for default products
  - `cup.png`
  - `sofa.png`
  - `t-shirt.png`

### Application Structure

#### App Directory (`src/app/`)

- **`layout.tsx`** - Root layout, contains side navigation bar (and page child components) wrapped with `EdgeStoreProvider`, `StoreProvider`, and `ThemeProvider`
- **`page.tsx`** - Root page (empty)
- **`globals.css`** - Global styles
- **`StoreProvider.tsx`** - Redux store provider

##### API Routes

- **`api/edgestore/[...edgestore]/route.ts`** - Simple file bucket with no validation

##### Dashboard Pages

- **`dashboard/orders/page.tsx`** - Orders page
- **`dashboard/products/page.tsx`** - Products and categories page

#### Components Directory (`src/components/`)

##### Navigation Components

- **`app-sidebar.tsx`** - Application sidebar
- **`nav-documents.tsx`** - Documents navigation
- **`nav-main.tsx`** - Main navigation
- **`nav-secondary.tsx`** - Secondary navigation
- **`nav-user.tsx`** - User navigation
- **`site-header.tsx`** - Site header
- **`theme-provider.tsx`** - Theme provider component

##### Data Table Components (`components/data-table/`)

- **`data-table.tsx`** - A reusable data-table component
- **`actions-dropdown-menu.tsx`** - Actions dropdown menu
- **`drawer-forms.tsx`** - Contains two forms for adding or editing products and categories
- **`table-taps.tsx`** - This is where data is selected from Redux store and then delivered to the tables

###### Table Columns

- **`columns/category-columns.tsx`** - Categories table columns
- **`columns/product-columns.tsx`** - Products table columns

##### UI Components (`components/ui/`)

Shadcn UI components including:
- Alert Dialog, Avatar, Badge, Breadcrumb
- Button, Card, Chart, Checkbox
- Drawer, Dropdown Menu, Form, Input
- Label, Select, Separator, Sheet
- Sidebar, Skeleton, Sonner, Table
- Tabs, Toggle Group, Toggle, Tooltip

##### Upload Components (`components/upload/`)

- **`dropzone.tsx`** - Dropzone component
- **`multi-image.tsx`** - Multi-image upload component
- **`progress-circle.tsx`** - Progress circle component
- **`uploader-provider.tsx`** - Uploader provider

#### Library Directory (`src/lib/`)

##### State Management

- **`store.ts`** - Redux store configuration, sets up the central state management store using Redux Toolkit, which manages the global application state. The store manages two main pieces of state:
  - Products (managed by `productsReducer`)
  - Categories (managed by `categoriesReducer`)

##### Redux Slices (`lib/features/`)

- **`categories/categories-slice.ts`** - Categories state slice
- **`products/products-slice.ts`** - Products state slice

##### Utilities

- **`utils.ts`** - Helper function from shadcn
- **`edgestore/edgestore.ts`** - Client-side configuration for EdgeStore, creates `useEdgeStore` hook used to interact with EdgeStore

##### Custom Hooks (`lib/hooks/`)

- **`hooks.ts`** - Redux helper hooks
- **`use-mobile.ts`** - Mobile detection hook
